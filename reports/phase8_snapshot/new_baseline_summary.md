# Phase 8: 新ベースライン評価レポート (リーク対策検証済み)

「タイムトラベル」ルールに厳密に準拠したパイプライン（`auto_predict_v13.py` バッチモード）を使用し、2025年の全予測データを再構築しました。
このベースラインは、オッズ取得時間を「発走10分前（T-10m）」に厳格に制限し、オッズ依存の特徴量をすべて上書きすることで、**データリークゼロ**を保証しています。

## 1. 検証方法 (Methodology)
- **推論実行**: `auto_predict_v13.py --year 2025 --batch`
- **オッズソース**: `apd_sokuho_o1` (SQL検索条件: `happyo_tsukihi_jifun <= T-10m`)
- **特徴量処理**: `preprocessed_data.parquet` (ベース) に対し、以下を上書き・削除:
    - **上書き**: `odds`, `tansho_odds` -> スナップショットオッズを使用
    - **上書き**: `popularity` -> スナップショットオッズに基づく順位で再計算
    - **上書き**: `prob_residual_softmax` -> スナップショットを用いて再計算
    - **削除**: `rank`, `payout` などの未来情報（Guardにて例外化済み）
- **ガードレール**:
    - 禁止列（`rank` 等）が予測機に入力された場合、`ValueError` を発生させる（`tests/test_feature_leak_guard.py` にてテスト済み）。
    - **フォールバック禁止**: 有効なT-10mオッズが存在しないレースは、予測自体をスキップ。

## 2. パフォーマンス結果 (Win EV > 1.0)

| 指標 (Metric) | 新ベースライン (No Leak) | プラセボ (Placebo / Random) |
| :--- | :--- | :--- |
| **ROI (回収率)** | **71.6%** | **69.9%** |
| 的中率 (Hit Rate) | 2.14% | 2.00% |
| 純損益 (Profit) | ¥-792,530 | ¥-836,570 |
| 平均購入件数 | ~8.5件 / レース | ~8.5件 / レース |

## 3. 分析と結論
- **リークの確定**: 修正前の高いROI（100%超）は、確定オッズや確定人気順のリークに依存していたことが確認されました。
- **真の実力値**: T-10m時点の情報に限定した場合、現在のv13モデルの予測能力はランダム（Placebo）よりわずかに高い程度（+1.7%）に留まります。これは、現在の特徴量セットにおいて「市場の歪み（Market Residual）」がT-10m時点ではほとんど捉えられていないことを示唆しています。
- **次のステップ**:
    - **信頼できる検証環境**: 今回の修正により、インフラは完全に信頼できるものとなりました。
    - **改善の方向性**: 今後のモデル改善（Phase 9以降）は、この「ROI 71.6%」を基準（ベースライン）として評価する必要があります。
    - **焦点の移動**: リークしていた「人気順」に頼らず、T-10m時点でも有効なシグナル（例: 直前投票の急激な変化、騎手変更の影響、より高度な前日特徴量など）の開発に注力する必要があります。
