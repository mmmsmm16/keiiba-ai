# プロジェクト現状確認回答

ご質問いただいた項目について、コードおよび最新のレポートから調査した結果は以下の通りです。

## 1. 狙っている馬券種
*   **主力:** **3連単 (Sanrentan)**
    *   フォーメーション: 1頭軸マルチなし（相手3~4頭、6~12点）
    *   戦略名: "Midrange", "Top3 Dominant", "Option C"
*   **ベース/抑え:** **馬連 (Umaren)**
    *   フォーメーション: 1頭軸流し（相手3頭、3点）
*   **回避時:** 単勝 (Option C) または 見送り

## 2. 学習の目的変数
*   **手法:** Learning to Rank (LightGBM `lambdarank`)
*   **ターゲット値 (Target Encoding):**
    *   1着 = **3**
    *   2着 = **2**
    *   3着 = **1**
    *   4着以下 = **0**
    *   (v13モデルでは 1.0/0.5/0.25 の浮動小数点グレードも試験実装されています)
*   **目的:** 勝ち馬だけでなく、3着内に入る馬の順序を正しく並べ替えること（NDCG最適化）。

## 3. オッズの使用について
*   **使用有無:** **あり** (予測・買い目判定の両方で使用)
*   **時点:**
    *   **バックテスト:** **確定オッズ (Final Odds)** を使用（レース後の最終結果）。
    *   **実運用 (推論):** 実行時点の**最新オッズ**を使用。
    *   **注意:** 現在の「リアルタイム連携機能 (Phase 18)」は未実装扱いのため、自動的に「発走N分前」を取得する仕組みではなく、ユーザーがスクリプトを実行した時点のオッズ（PC-KEIBAのDBにある最新値）が使われます。バックテスト結果は確定オッズに基づいているため、実運用より有利な数字が出ている可能性があります。

## 4. 直近のバックテスト結果
