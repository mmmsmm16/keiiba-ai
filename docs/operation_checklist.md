## 1. 資金管理ルール

### 初期設定

| 項目 | 推奨値 | 理由 |
|------|--------|------|
| **初期資金** | ¥100,000 | ペーパートレード実績ベース |
| **1レースあたり賭け金** | ¥400 (4枚×¥100) | BOX4 = 4組み合わせ |
| **1日あたり上限** | ¥12,000 (30レース相当) | 中央開催24-36R/日 |
| **月間上限** | ¥100,000 | 初期資金と同額 |

### 賭け金調整ルール

```
現在残高が初期の50%以下 → 賭け金を半額に
現在残高が初期の200%以上 → 賭け金を維持（増やさない）
```

## 2. リスク管理ルール

### MaxDD (最大ドローダウン) 設定

| 閾値 | アクション |
|------|------------|
| **-30% (¥30,000)** | ⚠️ 警告: 運用継続、賭け金半減 |
| **-50% (¥50,000)** | 🛑 停止: 1週間クールダウン |
| **-70% (¥70,000)** | ❌ 中止: 戦略再検討 |

### 連敗ルール

| 条件 | アクション |
|------|------------|
| 1日10連敗 | 当日残りレースをスキップ |
| 3日連続マイナス収支 | 翌日は賭け金半減 |
| 1週間連続マイナス | 運用一時停止、要レビュー |

## 3. 運用フロー

### 出走前 (レース30分前まで)

```
1. JRA公式サイトで出走確定を確認
2. paper_trade_run.py 実行 (予測生成)
3. 買い目レポート確認
4. 取消・除外馬をチェック
5. 買い目を控える or 購入手続き
```

### 購入時

```
1. JRAネット投票 or 競馬場窓口
2. レポートの買い目を入力
3. 金額確認 (¥100 × 4)
4. 購入確定
```

### レース後

```
1. 結果確認
2. paper_trade_settle.py 実行
3. 日次レポート確認
4. 残高記録
```

---

## 4. 例外処理

### データ欠損

| 状況 | 対応 |
|------|------|
| 当日データが取得できない | 当日の運用をスキップ |
| 特定レースのオッズ欠損 | そのレースをスキップ |
| 出馬表変更（取消等） | TOP4に取消馬がいればスキップ |

### システム障害

| 状況 | 対応 |
|------|------|
| 予測生成エラー | ログ確認、手動判断で見送り |
| ネット投票サービス障害 | 購入見送り |
| データベース接続エラー | 復旧後に再試行、無理なら見送り |

## 5. モニタリング指標

### 日次確認項目

| 指標 | 確認方法 | 警告閾値 |
|------|----------|----------|
| 当日ROI | daily_report.md | < 0% (マイナス) |
| 当日的中率 | daily_report.md | < 10% |
| 累積残高 | 手動記録 | -30% (MaxDD) |
| スキップ数 | run_summary.json | > 5レース/日 |

### 週次レビュー項目

| 指標 | 期待値 (ペーパー実績) | 警告閾値 |
|------|------------------------|----------|
| 週間ROI | 500%+ | < 100% |
| 週間的中率 | ~35% | < 20% |
| MaxDD | < 5% | > 15% |
| 総レース数 | 150+ | < 100 |

### 月次レビュー項目

- バックテスト ROI (612%) との乖離
- 週単位の収支推移
- 戦略調整の要否
